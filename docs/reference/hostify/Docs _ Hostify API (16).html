Introduction

Welcome to Hostify API.

Default base url

The default base URL for Hostify API is https://api-rms.hostify.com/

All data should be sent in JSON format and with a Content-Type: application/json header.

Note: for security reasons, all Hostify APIs are served over HTTPS only.

Global params

Parameter Type Values Default
include_related_objects int 0-1 0
page int positive 1
per_page int positive 20

Filter object

Parameter Description Values
field Subject of the filter
operator Comparison operator = <> < > <= >= in not_in between not_between
value Value to filter by

Operators in, not_in, between and not_between expects at least 2 comma-separated values.

Authentication

Hostify uses API keys to allow access to the API. You can register a new API key at our developer portal.
Hostify expects for the API key to be included in all API requests to the server in a header that looks like the following:
x-api-key: '<KEY>'

Get calendar

Get listing calendar

📋 Response Fields

Calendar responses include the following fields:

  • id - Calendar entry ID
  • date - Date in YYYY-MM-DD format
  • status - available, unavailable, or booked
  • price - Daily price
  • currency - Currency code (e.g., EUR, USD)
  • listing_id - Listing ID
  • reservation_id - Associated reservation ID if booked
  • min_stay - Minimum nights required
  • cta - Check-to-arrive: 1 (allowed), 0 (not allowed)
  • ctd - Check-to-depart: 1 (allowed), 0 (not allowed)
  • note - Calendar note
  • base_price - Base daily price before adjustments
  • is_manual_blocked - 1 if manually blocked, 0 otherwise
  • is_preparation_blocked - 1 if blocked for preparation, 0 otherwise
ℹ️ Note: Advanced pricing features (LOS, MBV, CTA/CTD overrides) are not enabled for your account. Contact support if you'd like to enable these features.

Bulk update calendar (multiple listings)

Update calendar for multiple listings in one request. Each listing can have different date ranges and settings.

Supported Fields

  • start_date - Start date (YYYY-MM-DD)
  • end_date - End date (YYYY-MM-DD)
  • price - Price per night
  • is_available - Availability (1 = available, 0 = unavailable, -1 = default)
  • note - Calendar note
  • min_stay - Minimum nights required
  • cta - Check-to-arrive (1 = allowed, 0 = not allowed, -1 = default)
  • ctd - Check-to-depart (1 = allowed, 0 = not allowed, -1 = default)

📝 Example: Update Multiple Listings

PUT /calendar/bulk_listings
Content-Type: application/json

[
  {
    "listing_id": 12345,
    "calendar": [
      {
        "start_date": "2025-12-20",
        "end_date": "2025-12-31",

        "end_date": "2025-12-31",
        "price": 200.00,
        "is_available": 1,
        "min_stay": 2

Assign thread

Assign or unassign a thread to a user. This endpoint updates the thread assignee in the core system.

Request Body

The request body should be sent as JSON with the following parameters:

  • thread_id (required): The ID of the thread to assign
  • assignee_id (optional): The user ID to assign the thread to. Pass null to unassign the thread.

Response

Returns the updated thread object with assignee details.

Assignee Object

The thread object includes an assignee field with the following structure:

  • id: User ID
  • first_name: User's first name
  • last_name: User's last name
  • name: Full name (first_name + last_name)
  • email: User's email address
  • avatar: URL to user's avatar image

Example: Assign thread to user

{
  "thread_id": 12345,
  "assignee_id": 29
}

Example: Unassign thread

{
  "thread_id": 12345,
  "assignee_id": null
}

Notes

  • The assignee must be an active user belonging to the same customer as the thread
  • The thread must exist in the core system (have a valid target_id)
  • Legacy threads without a target_id cannot be assigned
