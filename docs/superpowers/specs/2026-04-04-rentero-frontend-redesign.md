# Rentero Frontend Redesign — Design Spec
Date: 2026-04-04

## Overview

Full frontend redesign of the Rentero real estate portfolio management app. The goal is to replace the current light-shell / glassmorphism aesthetic with a "Dark SaaS Pro" design language — dense information design, premium data product feel, inspired by Linear/Vercel dashboards.

**Constraints:**
- Tech stack unchanged: FastAPI + Jinja2 + HTMX (server-rendered)
- No React, no npm, no build tooling
- All CSS in `base.html` as a single structured stylesheet (Variant 1)
- Tailwind CDN removed
- Sparklines: inline SVG generated by Jinja2
- Animations: CSS `@keyframes` + `transition` (no Framer Motion)
- Backend (`report/*.py`, `config/`) not touched

---

## Design Language

### Aesthetic
"Dark SaaS Pro" — dense information design, premium data product. Think Linear or Vercel dashboard meets real estate analytics. No pure glassmorphism (glassmorphism only for overlay panels and drawers). No flat minimalism.

### Animations

```css
/* Card mount stagger */
@keyframes fadeSlideUp {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: translateY(0); }
}
/* Applied via Jinja2 loop.index as --i */
.property-card { animation: fadeSlideUp 300ms ease both; animation-delay: calc(var(--i) * 40ms); }

/* Drawer slide */
.drawer { transform: translateX(100%); transition: transform 200ms ease-out; }
.drawer.open { transform: translateX(0); }

/* KPI count-up via CSS custom property + @property */
@property --num { syntax: '<integer>'; initial-value: 0; inherits: false; }
.kpi-value { transition: --num 800ms ease-out; }
```

## Screen: Přehled objektů (dashboard.html)

### Top Summary Bar
4 KPI cards in a row:
- Celková výplata klientům (sum CZK, green)
- Průměrná obsazenost (%, sparkline from last 6 months)
- Celkem rezervací (count + delta vs last month)
- Aktivní objekty (N / total)

### Filter Row
```
[← 04/2026 →]  [Vše] [S problémy ●N] [Nová data] [Uzavřeno]   [SVG-search-icon Hledat...]
```
- Month navigation: HTMX (existing `web.py` endpoints)
- Filter chips: JS toggles card visibility client-side (no reload)
- Search: JS filters card titles client-side

### Property Card Grid (3 columns)

```
┌────────────────────────────────┐
│ ● 28. Pluku 58  [Nová data]   │
│   28_Pluku_58 · Praha          │
│                                │
│ [████████████░░░] 78%          │
│                                │
│ 9 rez.   32,450 Kč   ×9 chyb  │
└────────────────────────────────┘
```

Occupancy bar: CSS `linear-gradient` with colored segments:
- Green = verified, amber = ke kontrole, red = error, gray = empty

Card states:
- Normal: `--bg-card` border
- Nová data: `border-left: 2px solid --color-teal` + `box-shadow: -4px 0 12px rgba(45,212,191,0.12)`
- S problémy: `border-left: 2px solid --color-red`
- Uzavřeno: `opacity: 0.6` + lock icon top-right
- Hover: `scale(1.008)`, `--border-active`, 80ms ease

Stagger animation: Jinja2 sets `style="--i: {{ loop.index0 }}"` on each card.

## Accessibility & Polish

- All interactive elements: focus ring `outline: 2px solid --color-primary; outline-offset: 2px`
- Keyboard nav: Tab through all actions, Escape closes drawer
- Loading states: skeleton rows (CSS animation, match exact row height)
- Empty states: centered icon + message + CTA button
- Tooltips: `title` attribute on all icon-only buttons

---

## Constraints & Rules

- No white/light theme
- No border-radius > 18px on cards
- No emoji in UI elements
- Font weights: 400 / 500 / 600 only (700 for KPI numbers only)
- No full-page modals — use drawers (right-slide) instead
- CSS variables for all colors (no hardcoded hex in component classes)
- No placeholder lorem ipsum text
